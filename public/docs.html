错误码

常见错误码

网关返回的错误结构为 { "error": { "message": "...", "code": "..." } }。

HTTP code 含义与排查
401 invalid_api_key API Key 无效、被禁用或未带 Authorization 头。检查 Key 是否正确、是否已启用。
405 method_not_allowed 请求方法不对。比如 /v1/models 需要 GET，/v1/responses 需要 POST。
403 endpoint_not_allowed 当前分组不允许访问该接口（models / responses / chat.completions）。
403 model_not_allowed 请求的模型不在你分组的可用清单内。用 GET /v1/models 查看可用模型。
403 stream_not_allowed 当前分组不允许流式。去掉 stream: true 重试。
403 max_output_tokens_exceeded 请求的输出 token 超过成员或分组上限。调小 max_tokens / max_output_tokens。
403 max_input_tokens_exceeded 请求体预估输入 token 超过成员或分组上限。缩短输入内容后重试。
403 access_denied Key 被封禁，或 Discord 账号未注册 / 不在类脑服务器 / 无公益站资源区权限。
402 quota_exceeded 当前 Discord 用户或 Key 的费用额度已用完。guest 默认按 Discord 用户共享 30 USD 额度。
402 request_cost_limit_exceeded 可选的单次费用上限被触发；当前站点默认关闭该限制，主要使用成员额度与 token 上限管理。
400 invalid_request 请求体不是合法 JSON，或缺少必要字段。
429 rate_limit_exceeded 请求过快。每个 Key 每分钟最多 4 次调用，超过后稍等一会儿再重试。
503 registration_store_unavailable 注册名单存储暂时不可用。稍后重试，或联系管理员检查 KV / Redis 配置。
502 upstream_request_failed 上游请求失败，稍后重试；持续失败请联系管理员。
💡 提示：每个 Key 的可用模型、是否允许流式、输入/输出 token 上限由其 成员设置或所属分组 决定。若调用被 403 拒绝，多半是策略限制，可联系管理员调整。
